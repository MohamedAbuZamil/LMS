# LMS Backend (Unified API)

Thin Fastify + Prisma + TypeScript backend that will serve all three frontends:

- Student Frontend → `lmszamil.azurewebsites.net`
- Teacher Frontend → `lmszamil-teacher.azurewebsites.net`
- Admin Frontend → `lmszamil-admin.azurewebsites.net`

> ⚠️ This is a **thin scaffold only** — no business logic, no auth routes yet. Just plumbing: Fastify server, Prisma client, CORS, health endpoint.

## Stack

- **Runtime:** Node 20 LTS
- **Framework:** Fastify 4
- **ORM:** Prisma 5 (introspected from existing PostgreSQL schema)
- **Language:** TypeScript (strict)

## Local development

```bash
cd backend
cp .env.example .env
# Edit .env and set DATABASE_URL with the real password
npm install
npm run prisma:pull       # introspect Azure DB into prisma/schema.prisma
npm run prisma:generate   # generate Prisma client
npm run dev               # http://localhost:8080/health
```

## Project structure

```
backend/
├── src/
│   ├── server.ts            # Fastify entrypoint (boot + listen)
│   ├── config/env.ts        # env loader
│   └── plugins/
│       └── prisma.ts        # Prisma client as Fastify decorator
├── prisma/
│   └── schema.prisma        # generated by `prisma db pull`
├── .env.example
├── package.json
└── tsconfig.json
```

## Routes

| Method | Path      | Description     |
| ------ | --------- | --------------- |
| GET    | `/`       | Service banner  |
| GET    | `/health` | Liveness probe  |

Business endpoints (`/api/auth/*`, `/api/admin/*`, `/api/teacher/*`, `/api/student/*`) will be added in subsequent iterations.
